車間制御 ACC — 前車に追従する MPC

前車との車間を一定に保つ。加速・減速には上限（快適性・安全の制約）。前車の急ブレーキにも先回りして減速し、追突を回避する。

モデル化

縦運動の物理 → 状態空間 → 離散化
自車が前車との車間を目標値 d* に保つ問題。状態は車間の誤差・相対速度・自車の加速度の3つで表す。前車の加減速は外乱として効く。

ae˙=u−aeτ 加速度の一次遅れ（τ）

制御ロジック（MPC）

予測＋制約付き最適化
MPC は未来を予測し、加減速の上限を守りながら車間誤差を最小にする加速度列を毎瞬間計算し、先頭だけ実行する。前車の急減速も先読みできる。
毎ステップ解く最適化（＋制約）
minu ∑k=1Np (qegap2+ru2) s.t. amin≤u≤amax
＝車間誤差 egap を小さく、かつ加減速 u を穏やかに。上限は制約として内蔵。
  • 未来 Np=25 ステップを予測して最適加速度列を計算、先頭だけ実行
  • 前車の急減速を先読みし、上限内で早めに減速
  • PIDとの違い：PIDは今の誤差にしか反応せず制約も知らない。MPCは未来と上限を同時に考える

天井クレーン アンチスウェイ — 揺らさず運ぶ MPC

吊り荷を載せたトロリーを目標位置へ運ぶ。台車に加える力は1入力だが、位置と吊り角の両方を同時に整える必要がある。MPCは振れを先読みし、到着時に残留スイングをほぼゼロにする。

倒立振子 cart-pole — 上向きの不安定平衡を立てる MPC

台車を1本の力 F で動かし、その上のフリーな振子を倒立させ続ける。本質的に不安定なプラントを、線形化モデルで未来を予測しながら安定化。力には上限（飽和）。横から蹴られても立て直す。

1 入力台車への水平力 F
姿勢 ＋ 台車位置を制御
1

モデル化

台車＋振子の力学 → 線形化 → 離散化
台車を押すと上の振子が倒れようとする不安定な連成系。状態は台車の位置 x・速度 ẋと振子の角度 θ・角速度 θ̇の4つ。入力は台車を押す力 F 1つだけで、倒立維持と台車の定位置保持を両立させたい。
運動方程式（台車＋振子・非線形 → θ≈0 で線形化）
x¨≈1M+mF−mgM+mθ ＝つまり台車は力Fで動き傾き θ に少し引かれる
θ¨≈(M+m)gL(43(M+m)−m)θ−1…F ＝つまり倒れるほどさらに倒れる（正係数＝不安定）。押すと逆へ傾く
→ 状態空間 ẋ = A x + B F　（x = [x, ẋ, θ, θ̇]ᵀ）
[x˙x¨θ˙θ¨] = [ 0100 00−1.400 0001 0016.820 ] [xx˙θθ˙] + [00.9520−1.429] F
左の縦ベクトル＝各状態の変化。2行目 ẍ・4行目 θ̈ が上の式そのもの。橙＝傾き θ の効き（+16.82＝倒れる側＝不安定）、青＝力 F の効き。実装は数値微分で A,B を算出。
離散化（ZOH・dt = 0.05 s）
xk+1=Adxk+BdFk Ad=eA·dt（数値ヤコビアンで A,B 構築）

4状態を罰する予測最適化
MPCは角度 θ を最も重く罰するコストで、未来 Np=40 ステップ（≒2秒）を予測し、力の加え方を整形。倒れる前に先回りして立て直し、同時に台車も中央へ戻す。線形なので事前計算ゲインで実行。

台車レース — 壁を守って目標へ走る MPC

台車を力で押して目標位置へ運ぶ。力には上限があり、前方には越えてはならない壁（位置の上限）がある。MPC は壁を制約として知り、手前で滑らかに止まる。

1 入力台車を押す力 F
制約付き MPC
1

モデル化

台車の運動 → 状態空間 → 離散化
台車を力 F で押して目標位置 r へ運ぶ。状態は台車の位置 x・速度 vの2つ。摩擦なしの2重積分系（力→加速度）で、前方には越えてはならない壁（位置の上限）がある。
運動方程式（ニュートンの法則・連続時間）
mx¨=F 摩擦なし。質量 m=1 と正規化 → 入力 u=F（加速度指令）、ẍ = u/m
x˙=v,v˙=u 位置 x と速度 v の 2 重積分系（力→加速度）
→ 行列形式 ẋ = A x + B u，y = C x　（x = [x, v]ᵀ）
[x˙v˙] = [ 01 00 ] [xv] + [01] u
1行目 ẋ=v、2行目 v̇=u がそのまま。紺＝速度が位置を動かす、青＝入力 u（力）が速度を動かす。出力 y=x（台車の位置）、C=[1, 0]。
離散化（ZOH・dt = 0.1 s）— 2重積分は厳密
Ad= [1dt01] Bd= [dt22dt]
xk+1=Adxk+Bduk　Ad=eA·dt, Bd=∫₀dteAsds·B（2重積分は打切りなしで厳密）
m = 1dt = 0.1 sx₀ = −2 m状態 2次入力 1次
2

制御ロジック（MPC）

予測＋制約付き QP
MPC は未来を予測し、力の上限を守りつつ壁を越えない位置列を毎瞬間計算して先頭だけ実行する。壁を制約として知るので、手前で早めに減力して滑らかに止まる。
毎ステップ解く最適化問題（追従＋省力）
minU ∑i=1Np (r−yk+i)2 +λ∑j=0Nc−1uk+j2
＝目標 r と予測位置 y の誤差を小さく、かつ力 u を穏やかに（重み λ）。
制約（守るべき上限）— QP の不等式
−Fmax≤uk+j≤Fmax 力の上限（飽和を最適化に内蔵）
yk+i≤xwall 位置の上限＝壁。予測位置が壁を越えない
  • 未来 Np=20 ステップを予測、制御 Nc=6 手を最適化し先頭だけ実行
  • QP は Hildreth 法で求解（重み λ=0.1、毎ステップ反復）
  • PIDとの違い：PID は壁を知らず突進して越える。MPC は壁を制約として持ち手前で減力

2D ドローン — ホバリング＆飛行する多入力 MPC

平面クアッドロータ。総推力と回転トルクの 2 入力で、水平位置・高度・姿勢を同時に制御。水平移動は機体を傾けて推力の横成分で行うため、位置と姿勢が連動する MIMO 問題になる。突風（横外乱）にも姿勢を立て直して復帰。

1

モデル化

平面の運動 → 状態空間 → 離散化
水平・高度・姿勢が連動する MIMO 系。状態は位置 x・高度 y・傾き θ とその速度の6つ、入力は総推力 δTとトルク u₂の2つ。水平に進むには機体を傾けて推力の横成分を使うので、位置と姿勢を切り離せない。
運動方程式（並進＝ニュートン・回転＝オイラー）
mx¨=−Tsinθ 推力の横成分で水平加速（傾けて進む）
my¨=Tcosθ−mg 縦成分−重力。総推力 T=mg+δT
Jθ¨=u2 左右ローター差＝トルクで機体を回転
→ 行列形式 ẋ = A x + B u　（x = [x, ẋ, y, ẏ, θ, θ̇]ᵀ, u = [δT, u₂]ᵀ）
[x˙x¨y˙y¨θ˙θ¨] = [ 010000 0000−g0 000100 000000 000001 000000 ] x+ [ 00 00 00 1/m0 00 01/J ] u
