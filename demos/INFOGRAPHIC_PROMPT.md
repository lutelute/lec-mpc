# 制御デモ「解説リッチ1枚インフォグラフ」作成プロンプト

制御工学の各デモ（ACC / 倒立振子 / クレーン / 台車レース / ドローン …）について、
**16:9 の 1 枚で「モデル化 → 制御ロジック → 制御の効果」が直感的に分かる**インフォグラフ HTML を作るための指示書。
数式は残しつつ、各ステップに「**言葉の解説（なぜ／何を意味するか）**」と図解を添え、
中級者が読んで理解できる“技術書的な分かりやすさ”を持たせる。

---

## 入力
- **テンプレート**: `demos/poster_acc.html`（CSS・レイアウト・配色・グラフ生成 JS の雛形）。クラス・配色・1280×720 固定・`@media print` をそのまま流用する。
- **対象デモ**: `demos/<name>.html`。ここから **物理モデル・状態方程式・制御則（MPC ゲイン計算）・PID・パラメータ・シミュレーションループ・ブロック線図 SVG** を抽出する。数値（行列・τ・質量・長さ・重み等）は必ずデモ実装と一致させる。

## 出力
- `demos/poster_<name>.html`（独立 HTML、1280×720 固定、印刷対応）

---

## レイアウト（横 3 列）
`ヘッダー（タイトル＋一言サブ＋右バッジ）` → `3 列` → `フッター（要点＋出典）`

### "解説リッチ" の肝 — 各列に必ず入れる 4 要素
1. **リード文（ひとことで言うと）**: 列の冒頭に 1〜2 行、平易な言葉で「この列で何を伝えるか」。淡色 callout ボックスで目立たせる。
2. **図（SVG）**: 概念図／ブロック線図／応答グラフ。重要点に**吹き出し・矢印の注釈**を付ける。
3. **数式（MathML）**: 残す。ただし各式の直後に「**＝つまり〜**」という平易な言い換え（`.eqnote`）を必ず添える。
4. **用語の注釈**: 初出の専門用語に短い説明を付ける（過剰にしない）。

### ① モデル化
- リード文「この系を何を使って・どんな状態量で表すか」
- プラント図（**変数を図に明示**：位置・速度・角度など）
- 運動方程式（連続時間の微分方程式を数本）＋ 各式が物理的に何を言っているか
- 状態空間 `ẋ = A x + B u`（**A, B を実値で**）＋「行列の各行が上の運動方程式に対応」という橋渡しの一言
- 離散化（dt 明記）＋ パラメータ chip

### ② 制御ロジック
- リード文「MPC は何をしているか（**未来予測＋制約付き最適化**）」を一言で
- ブロック線図（信号の流れに注釈：誤差 → 最適化 → 入力 → プラント → 出力）
- 最適化式 `min Σ(状態コスト + r·u²)` ＋「何を小さくしたいのか」の言葉
- 制約 ＋「守るべきルール（上限・壁など）」の説明
- **PID との違い**を 1 行（予測・制約を陽に扱えるか）

### ③ 制御の効果
- リード文「**このグラフの読み方**」（青 = MPC、橙 = PID、点線 = 制約／目標、網掛け = 外乱区間）
- 対象デモのシミュレーションエンジンを**流用して実行**した MPC vs PID の時系列（主要状態＋制御入力）＋ KPI 2 枚
- 「**だから何が言えるか**」の考察を 1〜2 行（MPC の優位を具体値で）

## 分かりやすさの工夫（チェックリスト）
- [ ] 各列に**リード文 callout**（淡い背景の囲み）
- [ ] 数式には必ず**平易な翻訳**（`.eqnote` の「＝つまり…」）
- [ ] 図に**吹き出し・矢印**で着目点を示す
- [ ] **色は役割で固定**：青 = MPC／主役、橙 = PID／比較、緑 = 制約 OK・目標、赤 = 制約・危険
- [ ] 専門用語は最小限＋短い注釈
- [ ] ③ に「読み方」と「結論」を言葉で

## 厳守事項
- **JS エラー 0**。③ は**実シミュレーションの実カーブ**（ダミー値・手書き波形は禁止）。
- モデルの数値は対象デモ HTML と**完全一致**（行列・パラメータをコピーして使う）。
- **各列が 720px 枠内**に収まること。MathML の実レンダリング高さは Node/DOM シムより大きいので、**各列 20px 以上の余裕**を見込む（行列 `mtable` は `rowspacing` を詰め、フォントを 11〜13px に）。
- 数式は MathML、文章は日本語・中級者向けトーン。

## 検証手順（親エージェント側で実施）
1. ローカルサーバーで配信し、ブラウザで `.poster` を 1280×720 で撮影。
2. 各 `.col` の `scrollHeight − clientHeight` を測り、**全列 0**（はみ出しなし）を確認。
3. 全グラフに MPC/PID 2 本のカーブが描かれ、**NaN なし**を確認。
4. リード文・数式翻訳・読み方ガイドが入っているか目視。

- 旧「数式重視の体系版」（`poster_*.html` の初版、Git 履歴に保存済み）を、この**解説リッチ版で上書き**する方針。
- もし数式版も別途残したい場合は、解説版を `poster_<name>_guide.html` 等の別名で出力する。
